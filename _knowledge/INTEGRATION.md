# Интеграция Knowledge Management System

> Этот файл читает Claude Code. Ты — Claude. Выполняй шаги последовательно, задавая пользователю вопросы когда нужно. Веди пользователя — он следует за тобой.

## Контекст

В папке `_knowledge/` лежит шаблон системы управления знаниями. Твоя задача — интегрировать его в текущий проект. Это значит: перенести файлы из `_knowledge/` в корень проекта, адаптировать под конкретный проект, и удалить `_knowledge/` после завершения.

**Архитектура системы:**
- **Project Brief ("Модель проекта")** в CLAUDE.md: сжатое описание проекта с принципами написания (HTML-комментарий в шаблоне). Brief — точка входа для любого агента.
- **Трёхуровневая адресация**: brief (что за проект) → roadmap (что делаем) → index (какие решения приняты)
- **Sessions split**: roadmap.md содержит только задачи и статусы, sessions.md — сессионный контекст (отдельные файлы)
- **Двухуровневые индексы**: hub `_index.md` (таблица доменов) + доменные `{domain}/_index.md` (детальный скелет)
- **Spoke-based AGENT_PROTOCOL**: hub-файл со ссылками на spoke-файлы (pipelines, specialists, context-packages, verification)
- **context.py**: скрипт для построения карт контекста из decisions (`/context CODE`, `/context CODE!`, `/context #tag`)
- **Финализированные хуки**: pre-commit проверяет обновление обоих уровней индексов, rebuild-index генерирует двухуровневую структуру

## Фаза 0: Разведка

Перед тем как задавать вопросы, изучи текущее состояние проекта:

1. Прочитай все файлы из `_knowledge/` — это шаблоны, которые ты будешь интегрировать.
2. Проверь, существует ли уже `CLAUDE.md` в корне проекта. Если да — прочитай его.
3. Проверь, существует ли `.claude/settings.local.json`. Если да — прочитай его.
4. Проверь, существует ли папка `meta/`. Если да — прочитай что в ней.
5. Проверь, существует ли папка `agents/`. Если да — прочитай что в ней.
6. Посмотри структуру проекта (ls корня, основные папки).
7. Посмотри git log (если есть) — пойми чем проект занимается.
8. **Определи тип установки:**
   - Если нет CLAUDE.md и нет meta/ → **FRESH INSTALL** (Фаза 1 → 2 → 3 → 4 → 5)
   - Если есть meta/decisions/ с доменными _index.md (hub+domain) → **UPGRADE v1.1→v2** (Фаза 1U → 2U → 3 → 4 → 5)
   - Если есть meta/decisions/ но БЕЗ доменных _index.md (монолитный _index) → **v1.0 detected**. Сообщи пользователю: "Обнаружена KM v1.0 (монолитные индексы). Для обновления до v2 нужно сначала мигрировать на v1.1. Рекомендация: удали текущие KM-файлы и запусти fresh install — это проще чем поэтапная миграция." Спроси: fresh install или прервать.
   - Если есть CLAUDE.md без KM-секций (нет "Инициализация", нет meta/) → **FRESH INSTALL** с merge CLAUDE.md

## Фаза 1: Интервью

Задай пользователю вопросы. Задавай ВСЕ за один раз, не по одному:

1. **Название проекта** — как называется?
2. **Описание** — что делает проект? (1-2 предложения)
3. **Фаза** — на какой стадии? (проектирование / активная разработка / поддержка / ...)
4. **Домены решений** — какие категории решений есть в проекте? Предложи свой вариант на основе того, что увидел в Фазе 0, и спроси подтверждение. Примеры:
   - Веб-приложение: `arch/`, `api/`, `ui/`, `db/`
   - ML-проект: `model/`, `data/`, `infra/`, `eval/`
   - Игра: `mechanics/`, `narrative/`, `tech/`, `art/`
5. **Домены исследований** — какие темы могут потребовать ресёрча? Предложи вариант.
6. **Теги** — предложи начальный набор хештегов на основе увиденного.
7. **Агенты** — нужны ли специализированные роли (типа "ревьюер", "тестировщик", "архитектор")? Или достаточно базовых (Архитектор + Планировщик)?
8. **Brief** — на основе ответов выше, напиши секцию "Модель проекта" для CLAUDE.md. Следуй принципам из HTML-комментария в шаблоне.

Если из Фазы 0 ты уже можешь уверенно ответить на какие-то вопросы — предложи свои варианты и спроси подтверждение вместо открытого вопроса. Экономь время пользователя.

## Фаза 1U: Интервью (upgrade)

Ты обновляешь существующую KM-систему. Большинство данных уже есть. Задай только:

1. **Что нового?** — какие значимые изменения произошли с момента установки?
2. **Brief** — есть ли секция "Модель проекта" в CLAUDE.md?
   - Если нет → нужно написать (Фаза 2U создаст)
   - Если да → проверить актуальность
3. **Sessions** — сессионный контекст в roadmap.md или в отдельном sessions.md?
   - Если в roadmap → нужно split (Фаза 2U выполнит)
   - Если отдельно → ОК
